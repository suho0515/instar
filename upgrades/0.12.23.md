# Instar v0.12.23 — WhatsApp Setup Bug Fixes & Regression Tests

## What Changed

Three critical WhatsApp setup bugs fixed with comprehensive regression tests:

1. **Baileys peer dependency resolution** — Replaced `require.resolve()` with dynamic `import()` for Baileys detection. `require.resolve()` fails in npx contexts because it resolves relative to the npx cache, not the user's project. Now tries both v6 (`@whiskeysockets/baileys`) and v7 (`baileys`) package names everywhere.

2. **Baileys 405 infinite reconnect loop** — When WhatsApp returns HTTP 405 (protocol version mismatch), the backend now logs a clear error with upgrade instructions and does NOT attempt to reconnect. Previously this caused an infinite reconnect loop since 405 errors never resolve by retrying.

3. **Dashboard QR polling silent failures** — The dashboard QR code polling now surfaces errors instead of silently returning null. Handles 401/403 with authentication failure messages, other HTTP errors with status codes, and network errors with connection failure messages.

Added 22 regression tests covering all three issues.
