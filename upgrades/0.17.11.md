# Upgrade Guide — v0.17.11

<!-- bump: patch -->

## What Changed

### Fix: better-sqlite3 native bindings not compiled in shadow install

When auto-update installs a new version to the shadow-install directory, it uses `--ignore-scripts` to prevent cloudflared's binary download from failing in environments where the download path doesn't exist. This also skipped better-sqlite3's native module build step, leaving no compiled `.node` binaries in the shadow install.

**Result**: TopicMemory and SemanticMemory both failed at startup on any agent using shadow install (introduced in v0.17.0), falling back to legacy JSONL-based behavior. The error message was "Could not locate the bindings file."

**Fix**: After each shadow install, `npm rebuild better-sqlite3` is now run to compile native bindings for the current Node.js version and platform. If the rebuild fails for any reason, the existing fallback behavior (legacy SQLite-less memory) remains active.

## What to Tell Your User

If your agent's memory has felt less capable recently (slower, no search, shorter context) after an auto-update, this patch restores full SQLite-backed memory. The agent will rebuild its knowledge graph and conversation memory on the next update cycle.

## Summary of New Capabilities

| Area | Change |
|------|--------|
| Shadow install | Rebuilds better-sqlite3 native bindings after installation |
| Affected components | TopicMemory, SemanticMemory |
| Fallback behavior | Unchanged — non-fatal if rebuild fails |
