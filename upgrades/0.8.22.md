# Upgrade Guide: Instar 0.8.22

## What Changed

### Agent Updates Topic — Separate Channel for Non-Critical Notifications
Previously, ALL notifications (updates, dispatches, memory pressure, quota alerts) went to the Agent Attention topic. Users expected "Attention" to mean something genuinely needs their attention — like a blocked task or critical error. Instead, it was cluttered with version update messages.

Now there are **two default topics**:

- **Agent Attention** — Critical items only. Blocked tasks, memory pressure, quota alerts, errors that need the user. The direct line for when you genuinely need them.
- **Agent Updates** — Informational. Version updates, dispatch notifications, feature announcements, upgrade notifications. Things the user should know but that don't require action.

The server auto-creates the Agent Updates topic on startup (blue icon, alongside the green Attention topic).

**What you should do:**
1. No action needed. The topic is created automatically on next server restart.
2. When you want to notify your user about something, ask: "Does this NEED their attention, or is it informational?" Route accordingly.
3. If you have existing code or CLAUDE.md that sends to the attention topic for non-critical things, update it to use the updates topic.

### Routing Changes
- **AutoUpdater** notifications now go to Agent Updates topic (was: Agent Attention)
- **AutoDispatcher** notifications now go to Agent Updates topic (was: Agent Attention)
- **Upgrade-notify sessions** (the "here's what's new" messages) now go to Agent Updates topic (was: Agent Attention)
- **Memory pressure alerts** stay on Agent Attention (critical — user may need to act)
- **Quota alerts** stay on Agent Attention (critical — may affect agent availability)
- **Sleep/wake detection** stays on Agent Attention (system health)

**What you should do:**
- No action needed. Routing is handled automatically by the server.

## What to Tell Your User

These are the user-facing highlights. Personalize based on your user's interests:

- **Cleaner notifications**: "I've separated my notification channels. The Attention topic is now only for things that genuinely need you — blocked tasks, critical errors, things I can't handle alone. Regular updates and version info go to a new Updates topic, so Attention stays signal-rich."
- **No more noise in Attention**: If your user complained about too many notifications in the Attention topic, acknowledge that and explain this is the fix.

## Summary of New Capabilities
| Capability | How to Use |
|-----------|-----------|
| Agent Updates topic | Auto-created on startup — informational notifications go here |
| Agent Attention topic | Now reserved for critical/blocking items only |
| Dynamic routing | AutoUpdater, AutoDispatcher, upgrade-notify all route to Updates automatically |
