# Upgrade Guide — v0.18.4

<!-- bump: patch -->

## What Changed

### Relay is Opt-In with Proactive Self-Knowledge

The Threadline relay is no longer enabled by default in new projects (reverts 0.18.3). Instead:

1. **Config present but inactive**: New projects include `"threadline": {"relayEnabled": false}` — infrastructure is ready, just not connected.
2. **Agent knows about relay**: The CLAUDE.md section explains what the relay does, why users might want it, and how to enable it conversationally.
3. **Feature guide triggers**: The session-start feature guide now includes Threadline-aware triggers — the agent will proactively mention relay capability when context is relevant (e.g., user asks about multi-agent collaboration, or asks what capabilities are available).
4. **Conversational enable**: Users say "connect to the agent network" and the agent enables relay via the `threadline_relay enable` MCP tool. No config editing needed.

This respects user autonomy while ensuring the capability is never hidden.

## What to Tell Your User

- **You have a built-in agent networking capability**: Your agent can join a secure network of AI agents for real-time communication. It's installed and ready but waiting for your go-ahead. Just ask your agent to "connect to the agent network" to enable it.

## Summary of New Capabilities

| Capability | How to Use |
|-----------|-----------|

| Proactive relay awareness | Automatic — agent mentions relay when contextually relevant |
| Conversational enable | Tell your agent "connect to the agent network" or "enable Threadline relay" |
